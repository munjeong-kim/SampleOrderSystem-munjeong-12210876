# 기능 명세 - 주문 승인/거절

## 개요
접수된 주문(`RESERVED`) 목록을 확인하고, 특정 주문에 대해 승인 또는 거절을 처리한다.

## 담당자
생산 담당자

## 하위 기능

### 1. 접수된 주문 목록
`RESERVED` 상태의 주문 목록을 표시한다.

### 2. 주문 승인
접수된 특정 주문에 대해 승인을 처리한다. 승인 시 **재고 상황에 따라 2가지 방식으로 자동 분기**된다.

| 조건 | 처리 |
|---|---|
| 재고가 충분한 경우 | 주문을 즉시 `CONFIRMED` 상태로 전환 |
| 재고가 부족한 경우 | 생산 라인에 자동으로 등록하고, 주문 상태를 `PRODUCING`으로 전환 |

재고 부족 시 부족분만 생산한다. 부족분 계산 및 실생산량 산출 방식은
[05-production-line.md](05-production-line.md)를 참고한다.

### 3. 주문 거절
접수된 특정 주문에 대해 거절을 처리한다. 처리 즉시 `REJECTED` 상태로 전환된다. 거절 사유는
별도로 입력받거나 기록하지 않는다.

## 상태 전이

```
RESERVED
  ├─ 승인 + 재고 충분 → CONFIRMED
  ├─ 승인 + 재고 부족 → PRODUCING (생산 라인 큐 등록)
  └─ 거절            → REJECTED
```

## 비즈니스 규칙
- 승인/거절 대상은 `RESERVED` 상태의 주문만 가능하다.
- `REJECTED` 상태가 된 주문은 이후 어떤 프로세스로도 진행되지 않는다(모니터링에서도 유효 주문으로 집계하지 않음).
- 재고 충분/부족 판단 기준: 주문 수량 ≤ 현재 재고 → 충분, 주문 수량 > 현재 재고 → 부족.
