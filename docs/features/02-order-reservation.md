# 기능 명세 - 시료 주문 (주문 접수)

## 개요
고객이 시료를 요청하면 **주문 담당자**가 주문을 생성(접수)한다.

## 담당자
주문 담당자

## 처리 흐름
1. 주문 담당자가 원하는 시료와 수량을 입력해 주문을 생성한다.
2. 등록되지 않은 시료 ID는 주문할 수 없다 ([01-sample-management.md](01-sample-management.md) 참고).
3. 주문이 생성되면 상태는 `RESERVED`로 등록된다.

## 예약 시 입력 값
| 항목 | 설명 |
|---|---|
| 시료 ID | 주문할 시료의 ID |
| 고객명 | 주문 고객 이름 |
| 주문 수량 | 주문 수량(ea) |

## 결과
- 주문번호가 발급된다. (예: `ORD-20260416-0043`)
- 주문 상태: `RESERVED`
- 이 시점에는 재고 확인을 하지 않는다. 재고 확인 및 승인/거절은
  [03-order-approval.md](03-order-approval.md) 단계에서 처리한다.

## 비즈니스 규칙
- 주문 생성 시점에는 재고 부족 여부를 검증하지 않는다(승인 단계에서 검증).
- 주문 접수만으로는 재고가 차감되지 않는다.
