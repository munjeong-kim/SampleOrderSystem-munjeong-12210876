# 기능 명세 - 생산 라인

## 개요
생산 라인에 대한 정보를 표시한다. 주문량에 대한 부족분을 생산하되, 수율 및 오차를 고려하여
시료를 생산한다.

## 생산 라인 정책
- 생산 라인은 **1개(단일 라인)** 만 존재한다. 동시에 하나의 주문만 생산할 수 있다.
- 생산 대기열(생산 큐)은 **FIFO(선입선출)** 전략을 따른다. 먼저 큐에 들어온 주문부터 순서대로
  생산하며, 임의로 순서를 건너뛰거나 새치기할 수 없다.
- 시료의 초기 재고는 **0**이다. 재고는 시료 등록 시 입력받지 않으며, 오직 생산 완료를 통해서만
  쌓인다.
- 생산은 **실시간 대기** 방식으로 처리한다. 즉, 총 생산 시간만큼 실제 시간이 경과해야 생산이
  완료된 것으로 처리한다(즉시 완료 처리하지 않는다).

## 생산량 계산

| 항목 | 계산식 |
|---|---|
| 부족분 | 주문량 - 재고 |
| 실생산량 | `ceil(부족분 / 수율)` |
| 총 생산 시간 | 평균 생산시간 × 실생산량 |

예) 주문량 200ea, 재고 30ea, 수율 0.92
- 부족분 = 200 - 30 = 170ea
- 실생산량 = ceil(170 / 0.92) = 185ea (수율에 따른 오차 보정)

## 처리 흐름
1. [03-order-approval.md](03-order-approval.md)에서 재고 부족으로 승인된 주문이 생산 큐에 FIFO
   순서로 등록된다(주문 상태 `PRODUCING`).
2. 생산 라인은 큐의 가장 앞 주문부터 순서대로 생산을 진행하며, 총 생산 시간만큼 실시간으로
   대기한다.
3. 생산이 완료되면 실생산량 전량이 해당 시료의 재고로 편입되고, 주문 상태가 `PRODUCING` →
   `CONFIRMED`로 변경된다.
4. 생산 라인은 큐의 다음 주문을 이어서 생산한다.

## 하위 기능

### 생산 현황 표기
현재 생산 중인 시료에 대한 정보를 표시한다. (표기 정보 수준은 자율 결정: 주문 정보, 현재까지의
생산량 진행률 등)

### 대기 주문 확인
생산 큐에서 생산 작업을 대기하고 있는 목록을 FIFO 순서대로 출력한다.

## 비즈니스 규칙
- 생산 라인은 1개이므로 동시에 2개 이상의 주문을 생산할 수 없다.
- 생산 큐는 FIFO로만 처리되며, 우선순위 변경/새치기는 허용하지 않는다.
- 부족분·실생산량 계산은 항상 승인 시점의 재고를 기준으로 한다.
- 시료의 초기 재고는 0이며, 생산 완료 시 실생산량 전량이 재고로 편입된다.
- 생산은 실시간 대기 방식으로 처리하며, 총 생산 시간이 지나야 완료로 처리한다.
