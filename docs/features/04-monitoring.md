# 기능 명세 - 모니터링

## 개요
주문 및 시료 재고에 대한 정보를 모니터링하는 기능. 담당자가 현재 시스템의 상태를 한눈에
파악할 수 있도록 구성한다.

## 하위 기능

### 1. 주문량 확인
현재 상태별(`RESERVED` / `CONFIRMED` / `PRODUCING` / `RELEASE`) 주문 목록(건수)을 확인한다.

- `REJECTED`는 유효한 주문이 아니므로 집계에서 제외한다.

### 2. 재고량 확인
각 시료별 현재 재고 수량을 확인한다. **`RESERVED` 상태 주문 기준 필요 수량 대비** 재고 수량에
따라 상태를 함께 표기한다.

**필요 수량 산출**
```
필요 수량(시료별) = 해당 시료를 주문한 RESERVED 상태 주문들의 수량 합
```

| 상태 | 기준 |
|---|---|
| 고갈 | 재고 == 0 |
| 부족 | 0 < 재고 < 필요 수량 |
| 여유 | 재고 > 0 이고 재고 >= 필요 수량 |

## 비즈니스 규칙
- `REJECTED` 주문은 어떤 통계/집계에도 포함하지 않는다.
- 재고 상태 판단 시 "필요 수량"은 `RESERVED` 상태 주문만 집계한다. 이미 `PRODUCING`/`CONFIRMED`로
  전환된 주문은 승인 시점에 재고 반영이 끝난 것으로 보고 제외한다.
- 재고가 0이면 필요 수량과 무관하게 항상 "고갈"로 표기한다.
- 특정 시료에 대해 `RESERVED` 주문이 없으면 필요 수량은 0으로 간주하고, 재고가 0보다 크면
  "여유"로 표기한다.
